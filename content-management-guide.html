How to Update Website Content

Since this is a static website hosted on Netlify, content updates are managed through GitHub. Follow these steps:

  1. Access Your Repository - Go to your GitHub repository for the AGA Club website
  2. Navigate to content.json - Find and open the content.json file
  3. Edit Content - Click the pencil icon to edit the file directly in GitHub
  4. Make Changes - Update the relevant sections (see examples in other tabs)
  5. Commit Changes - Add a descriptive commit message and commit directly to the main branch
  6. Wait for Deployment - Netlify will automatically rebuild and deploy your site (1-2 minutes)

Important Notes

  • All content is stored in the content.json file
  • Images should be uploaded to the images/ folder
  • Always use valid JSON syntax
  • Multiple admins can collaborate on content updates

Need Help?

Refer to the detailed GitHub_Workflow_Guide.md file in your repository for comprehensive instructions.

Upcoming Events Format

To update upcoming events, edit the "upcomingEvents" section in content.json:

{
  "upcomingEvents": [
    {
      "id": 1,
      "name": "Event Name",
      "date": "Event Date",
      "venue": "Event Venue",
      "description": "Event Description",
      "formLink": "https://forms.google.com/...",
      "image": "images/event-image.jpg"
    }
  ]
}

Adding New Events

  1. Increment the ID for the new event (ensure each event has a unique ID)
  2. Fill in the event name, date, venue, and description
  3. Add an optional registration form link if applicable
  4. Specify an image from the images directory
  5. Save the content.json file

Automatic Removal of Past Events

The system automatically removes past events from the upcoming events section once their event date has passed. This is handled by the JavaScript code in upcoming.html which compares event dates with the current date.

Moving Events to Past Events

When an event date has passed, the event is automatically moved from the upcomingEvents section to the pastEvents section:

  1. The event is removed from the upcomingEvents array
  2. The event is added to the pastEvents array with the same ID
  3. The formLink field is removed as it's no longer needed

Best Practices

  • Regularly check that event dates are accurate
  • Use consistent date formatting (e.g., "Month Day, Year")
  • Ensure all image paths are correct and images exist
  • Test the website after making changes to verify events display correctly

Note: Always increment the ID for new events and ensure all entries are valid JSON.

Event Directories & Slideshow Images

The Events page displays slideshow galleries for each event by scanning event directories in the images/ folder. Each event directory should contain numbered images (1.jpg, 2.jpg, etc.) for the slideshow.

Adding New Event Directories

  1. Create a new folder in the images/ directory with a descriptive name for your event
  2. Name the folder using clear, descriptive text (e.g., "Annual Sports Day 2025")
  3. Add images to the folder with numeric filenames (1.jpg, 2.jpg, 3.jpg, etc.)
  4. Number images sequentially from 1 to maintain proper slideshow order
  5. Upload up to 20 images per event (the system checks for images 1-20)

Updating Event Directory Information in content.json

Edit the "eventDirectories" section in the "eventsContent" of content.json to include your new event:

{
  "eventsContent": {
    "eventDirectories": [
      { 
        "dir": "images/Your Event Folder Name", 
        "name": "Display Name for Your Event" 
      }
    ]
  }
}

Ensuring Events Load Properly in All Environments

To ensure events load correctly when the HTML files are opened directly in the browser, you must maintain complete event information in the content.json file:

For Past Events

Each past event in the "pastEvents" section should include:

  • Unique ID number
  • Event name
  • Event date
  • Description
  • Main event image path
  • Complete "slideshowImages" array with all image paths for the event gallery

Sample Past Event Entry

{
  "id": 1,
  "name": "Event Name",
  "date": "Event Date",
  "description": "Event Description",
  "image": "images/main-event-image.jpg",
  "slideshowImages": [
    "images/event-folder/1.jpg",
    "images/event-folder/2.jpg",
    "images/event-folder/3.jpg"
  ]
}

Cross-Environment Compatibility

The website works in three different environments:

  1. GitHub Deployment: Content loaded from content.json
  2. Direct Browser File Access: Content loaded from content.json
  3. Local Server: Content loaded from directory scanning (fallback)

For direct browser access to work properly:

  • All image paths in content.json must be relative to the website root
  • Every past event must have a complete "slideshowImages" array
  • All referenced images must exist in the specified locations
  • Test the website by opening HTML files directly in the browser after making changes

Thumbnail Hover Slideshow Functionality

The thumbnail hover slideshow functionality displays images from each event's dedicated image folder when users hover over event thumbnails. To ensure this works correctly:

  • The "slideshowImages" array in each past event must contain valid paths to all images for that event
  • Image paths must be correctly formatted and accessible
  • Up to 5 images from the slideshowImages array will be displayed in the hover slideshow
  • If slideshowImages is missing or empty, the hover functionality will not be enabled for that event

Best Practices for Event Images

  • Use high-quality JPG images with good lighting
  • Resize images to appropriate dimensions (recommended: 1920x1080 pixels)
  • Use descriptive filenames for easy identification
  • Ensure all images in an event folder are consistently sized
  • Number images sequentially to control slideshow order
  • Include a variety of shots (wide angles, close-ups, candid moments)

Troubleshooting

If event slideshows aren't displaying:

  • Check that the event directory exists in the images/ folder
  • Verify that the directory name matches exactly in content.json
  • Ensure images are named with numeric filenames (1.jpg, 2.jpg, etc.)
  • Confirm that images are valid JPG files
  • Test the website after making changes
  • Verify that all slideshowImages paths in content.json are correct
  • Check browser console for any error messages

Image Renamer Tool

The Image Renamer tool helps you organize event photos by renaming multiple JPG files with a consistent naming pattern.

How to Use:

  1. Open rename_images.html in your browser
  2. Enter a prefix for your images (e.g., "diwali", "holi", "event")
  3. Select multiple JPG files you want to rename
  4. Preview the renamed files
  5. Download the renamed files

Naming Convention:

The tool automatically numbers your files sequentially:

  • First file: [prefix]1.jpg
  • Second file: [prefix]2.jpg
  • Third file: [prefix]3.jpg
  • And so on...

Example:

If you enter "diwali" as prefix and select 3 images:

  • Original files might be: IMG_001.jpg, DSC_002.jpg, photo3.jpg
  • Renamed files will be: diwali1.jpg, diwali2.jpg, diwali3.jpg

Best Practices:

  • Use descriptive prefixes that match your event names
  • Create a new folder for each event before renaming files
  • Always preview files before downloading to confirm naming
  • Keep backups of original files before renaming

Filename Extractor Tool

The Filename Extractor tool helps you extract filenames with extensions from a list of file paths and format them for use in content.json or other purposes.

How to Use:

  1. Open extract_filenames.html in your browser
  2. Either:
    • Paste a list of file paths in the input area (one per line), OR
    • Select multiple files using the file selector (this automatically extracts filenames)
  3. Click "Extract Filenames" to get just the filenames with extensions
  4. Optionally add a prefix or suffix to each filename
  5. Click "Copy to Clipboard" to copy the results
  6. Use "Join with Comma" to create a comma-separated list for JSON arrays

Features:

  • File Selection: Select multiple files directly from your computer to automatically extract their names
  • Extract Filenames: Get just the filename with extension from full paths
  • Prefix/Suffix: Add text before or after each filename
  • Copy to Clipboard: Easily copy results for use elsewhere
  • Join with Comma: Format filenames as a comma-separated list for JSON arrays
  • Clear All: Reset all fields to start fresh

Example Usage:

If you have a list of image paths like:

images/events/diwali/IMG_001.jpg
images/events/diwali/DSC_002.jpg
images/events/diwali/photo3.jpg

The tool will extract:

IMG_001.jpg
DSC_002.jpg
photo3.jpg

With prefix "diwali" and suffix "":

diwaliIMG_001.jpg
diwaliDSC_002.jpg
diwaliphoto3.jpg

Joined with comma:

diwaliIMG_001.jpg, diwaliDSC_002.jpg, diwaliphoto3.jpg

Best Practices:

  • Use this tool when creating "slideshowImages" arrays in content.json
  • Always verify extracted filenames match your actual files
  • Use the prefix feature to standardize naming conventions
  • Join with comma for easy insertion into JSON arrays

Special Positions
